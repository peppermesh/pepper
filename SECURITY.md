# Security Policy

## Reporting a Vulnerability

Please do not report suspected vulnerabilities through public GitHub issues.

Report vulnerabilities privately through GitHub's private vulnerability reporting feature for this repository.

Include, when available:

- affected component;
- affected version or commit;
- reproduction steps;
- impact;
- suggested mitigation;
- whether the issue is already public.

## Scope

Security reports may include:

- remote code execution;
- sandbox escape;
- unauthorized block access;
- identity or signature bypass;
- replication integrity failures;
- privilege escalation;
- denial-of-service vulnerabilities;
- secret exposure;
- dependency compromise;
- unsafe default configuration.

## Licensing and Provenance Reports

Suspected unauthorized copying, incompatible licensing, or accidental inclusion of proprietary code should also be reported privately to the maintainers rather than through a public issue.
